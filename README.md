# แนะให้แนว — เว็บกิจกรรมแนะแนว ม.1–ม.6

เว็บไซต์รวมกิจกรรม แบบทดสอบ และเกมแนะแนว จัดกลุ่มตามระดับชั้น ม.1–ม.6
ตอนนี้มีเนื้อหาพร้อมใช้ที่ **ม.6** (12 กิจกรรม) ส่วน ม.1–ม.5 เป็นหน้า "เร็วๆ นี้" รอเพิ่มเนื้อหา

## โครงสร้างไฟล์
```
index.html            หน้าแรก — เส้นทาง ม.1–ม.6 (วาดอัตโนมัติ)
assets/style.css       ธีมกลาง ใช้ร่วมกันทุกหน้า (สี, ฟอนต์, การ์ด)
assets/activities.js    ★ ไฟล์เดียวที่ต้องแก้เวลาเพิ่มกิจกรรม ★
assets/site.js          ตัววาดหน้าเว็บ (ไม่ต้องแก้)
m1/ ... m6/index.html   หน้ารวมกิจกรรมแต่ละชั้น (วาดอัตโนมัติ ไม่ต้องแก้)
m6/*.html               ไฟล์กิจกรรมทั้ง 12 ไฟล์ (ของเดิม เปลี่ยนแค่ชื่อไฟล์เป็นอังกฤษ)
```

## วิธีเพิ่มกิจกรรมใหม่ (ง่ายที่สุด — ไม่ต้องแตะ HTML เลย)

**ขั้นตอน:**
1. เอาไฟล์ `.html` ของกิจกรรมใหม่ ไปวางในโฟลเดอร์ของชั้นนั้น เช่น `m5/career-quiz.html`
   (ตั้งชื่อไฟล์เป็นอังกฤษ ไม่มีเว้นวรรค/ภาษาไทย)
2. เปิดไฟล์ `assets/activities.js` แล้วคัดลอกกลุ่มนี้ 1 ชุด มาวางต่อท้ายในลิสต์:
   ```js
   { grade: 5, category: "ชื่อหมวด", file: "career-quiz.html",
     title: "ชื่อกิจกรรมที่จะโชว์บนการ์ด",
     desc: "คำอธิบายสั้นๆ 1 บรรทัด" },
   ```
3. แก้ 5 ค่า: `grade` (เลขชั้น 1-6), `category` (พิมพ์ชื่อหมวดเดิมซ้ำ = เข้าหมวดเดิม / พิมพ์ชื่อใหม่ = ระบบสร้างหมวดใหม่ให้อัตโนมัติ), `file`, `title`, `desc`
4. บันทึกแล้วอัปโหลดขึ้น GitHub (ดูวิธีด้านล่าง)

**เท่านี้จบ** — หน้าแรกจะรู้เองว่า ม.5 มีกิจกรรมแล้ว, หน้า ม.5 จะขึ้นการ์ดใหม่ให้เอง, ถ้าใช้ชื่อหมวดใหม่ก็จะแยกหมวดให้เองด้วย ไม่ต้องไปเปิด index.html ของหน้าไหนแก้เพิ่มอีก

## วิธีอัปโหลดไฟล์ขึ้น GitHub แบบไม่ต้องใช้ terminal (ง่ายสุด)
1. ไปที่หน้า repo บน GitHub → เข้าโฟลเดอร์ที่ต้องการ (เช่น `m5/`)
2. กด **Add file → Upload files** → ลากไฟล์ `.html` ใหม่วางแล้ว **Commit changes**
3. กลับไปที่ `assets/activities.js` → กดไอคอนดินสอ (Edit) มุมขวาบน → เพิ่มข้อความตามขั้นตอนด้านบน → **Commit changes**
4. รอ 1–2 นาที เว็บจะอัปเดตเองที่ลิงก์ GitHub Pages เดิม

## วิธีอัปโหลดขึ้น GitHub ผ่าน terminal (สำหรับตอนตั้งค่าครั้งแรก)

**ครั้งแรก (สร้าง repo ใหม่):**
1. ไปที่ github.com → New repository → ตั้งชื่อ เช่น `naegivnaew` → Create repository (ไม่ต้องติ๊ก README)
2. ในเครื่องที่มีโฟลเดอร์นี้ เปิด terminal แล้วรันตามนี้ (แทน `YOUR-USERNAME`):
   ```
   cd naegivnaew-site
   git init
   git add .
   git commit -m "เว็บกิจกรรมแนะแนว ม.1-6 เริ่มต้น"
   git branch -M main
   git remote add origin https://github.com/YOUR-USERNAME/naegivnaew.git
   git push -u origin main
   ```
3. ไปที่หน้า repo บน GitHub → Settings → Pages
4. หัวข้อ "Build and deployment" → Source เลือก **Deploy from a branch** → Branch เลือก **main** / โฟลเดอร์ **/(root)** → Save
5. รอ 1–2 นาที เว็บจะขึ้นที่ `https://YOUR-USERNAME.github.io/naegivnaew/`

**ครั้งต่อไป (อัปเดตเนื้อหา):** แก้ไฟล์ในเครื่อง แล้วรัน
```
git add .
git commit -m "เพิ่มกิจกรรม ม.5"
git push
```
เว็บจะอัปเดตเองภายใน 1–2 นาที
